# Ejercicio 1: Gestión de Notas por Asignatura (Matrices)

## 📝 Descripción
Una universidad necesita un sistema para gestionar las calificaciones de un grupo de estudiantes en diferentes asignaturas. Debes desarrollar un programa que solicite la cantidad de alumnos y la cantidad de asignaturas. El programa debe almacenar las notas en una **matriz** y luego generar un reporte que incluya:
1. El promedio final de cada alumno (promedio por fila).
2. El promedio general de cada asignatura (promedio por columna).
3. La nota más alta detectada en todo el curso.

> **Contexto Académico**: Este ejercicio profundiza en el recorrido de arreglos bidimensionales utilizando ciclos anidados y el manejo de índices para acceder a filas y columnas de forma independiente.

## 🎯 Objetivos de Aprendizaje
- Declaración y manipulación de matrices (`double[][]`).
- Implementación de recorridos por fila y por columna.
- Uso de acumuladores específicos para diferentes dimensiones de la matriz.
- Búsqueda de valores extremos (máximos) en una estructura bidimensional.

## 📥 Ejemplo de Entrada
```text
Cantidad de alumnos: 2
Cantidad de asignaturas: 3

Notas Alumno 1:
- Asignatura 1: 5.5
- Asignatura 2: 4.0
- Asignatura 3: 6.2

Notas Alumno 2:
- Asignatura 1: 4.5
- Asignatura 2: 3.5
- Asignatura 3: 5.0
```

## 📤 Ejemplo de Salida
```text
=== REPORTE ACADÉMICO ===
Promedio Alumno 1: 5.23
Promedio Alumno 2: 4.33

Promedio Asignatura 1: 5.00
Promedio Asignatura 2: 3.75
Promedio Asignatura 3: 5.60

Nota Máxima del Curso: 6.2
```
